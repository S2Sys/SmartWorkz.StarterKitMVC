HOW TO DEPLOY
================================================================================

OPTION 1: PowerShell (Recommended - Automated)
==========================================

1. Open PowerShell as Administrator

2. Navigate to database folder:
   cd C:\path\to\SmartWorkz.StarterKitMVC\database

3. Run deployment script:
   .\Deploy-Database.ps1 -ServerName "localhost" -DatabaseName "SmartWorkz_v4" -IntegratedSecurity

4. Wait for success message

Option: Using SQL Server Authentication instead of Windows Authentication:
   .\Deploy-Database.ps1 -ServerName "localhost" -DatabaseName "SmartWorkz_v4" -Username "sa" -Password "YourPassword"

Option: Using Named Instance (e.g., SQLEXPRESS):
   .\Deploy-Database.ps1 -ServerName "MYCOMPUTER\SQLEXPRESS" -DatabaseName "SmartWorkz_v4" -IntegratedSecurity


OPTION 2: SQL Server Management Studio (Manual)
==============================================

1. Open SQL Server Management Studio (SSMS)

2. Create new database:
   - Right-click Databases → New Database
   - Name: SmartWorkz_v4
   - Click OK

3. Execute scripts in order:
   - Right-click SmartWorkz_v4 → New Query
   - Copy content from 001_InitializeDatabase.sql
   - Execute (F5)
   - Repeat for 002-008 in order (IMPORTANT: Execute in sequence!)


OPTION 3: Batch File (Alternative Automation)
=============================================

1. Edit deploy.bat and set:
   - SERVER=localhost
   - DATABASE=SmartWorkz_v4
   - USERNAME=sa
   - PASSWORD=YourPassword

2. Run: deploy.bat

TROUBLESHOOTING
================================================================================

ERROR: "Cannot open database 'SmartWorkz_v4'"
SOLUTION: Database needs to exist first. PowerShell script creates it automatically,
          or create manually in SSMS: CREATE DATABASE SmartWorkz_v4;

ERROR: "The target database contains a schema 'Master'"
SOLUTION: Schema already exists. Drop database and retry:
          DROP DATABASE SmartWorkz_v4;

ERROR: "Login failed for user 'sa'"
SOLUTION: 1) Verify SQL Server is running
          2) Verify sa password is correct
          3) Verify SQL Server Authentication is enabled
          4) Restart SQL Server if needed

ERROR: "Timeout expired"
SOLUTION: Some tables take long to create. Use -IntegratedSecurity flag
          or edit Deploy-Database.ps1 to increase timeout:
          $sqlCommand.CommandTimeout = 600 (10 minutes instead of 5)

ERROR: "Cannot find path" in PowerShell
SOLUTION: Make sure you're in the correct directory:
          cd C:\path\to\SmartWorkz.StarterKitMVC\database
          .\Deploy-Database.ps1 ...

ERROR: "File not found" when running script
SOLUTION: Ensure all 8 SQL files are in the same folder as Deploy-Database.ps1
